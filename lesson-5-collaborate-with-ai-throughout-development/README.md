# Lesson 5 Exercises: Collaborate with AI Throughout the Development Cycle

This lesson contains hands-on exercises that guide you through collaborating with AI in every phase of software development. You'll work through the complete development cycle of a CLI flashcard quizzer, experiencing how to maintain engineering standards while leveraging AI assistance.

## Exercise Structure

Each concept focuses on a specific phase of the development cycle:

1. **Concept 1: AI Planning Collaboration** - Learn to use AI for architectural planning while maintaining engineering ownership
2. **Concept 2: AI Code Generation** - Generate high-quality, maintainable code with AI assistance
3. **Concept 3: AI Test Creation** - Build comprehensive test suites through AI collaboration
4. **Concept 4: AI Refactoring Collaboration** - Improve code quality with AI-assisted refactoring
5. **Concept 5: AI Documentation Generation** - Create professional documentation with AI support

## Learning Objectives

By completing these exercises, you will:

- Apply systematic engineering judgment to AI-assisted development
- Maintain architectural control while leveraging AI implementation capabilities
- Generate production-ready code through effective AI collaboration
- Build comprehensive test suites that validate both functionality and quality
- Refactor code systematically while preserving correctness
- Create professional documentation that serves real user needs

## Exercise Instructions

1. **Start with the `starter/` folder** for each concept
2. **Follow the step-by-step instructions** in each concept's README
3. **Run the provided tests** to validate your implementation
4. **Compare with the `solution/` folder** to see recommended approaches
5. **Experiment with different AI tools** using the provided structured XML prompts

## Prerequisites

- Python 3.8+
- Basic familiarity with an AI assistant (ChatGPT, Claude, GitHub Copilot, etc.)
- Understanding of software engineering principles from previous lessons
- Completion of Lesson 3 (structured XML prompting techniques)

## Getting Started

```bash
# Navigate to the first exercise
cd concept1-ai-planning-collaboration/starter

# Follow the README instructions for each concept
# Run tests to validate your progress
pytest test_*.py -v
```

## Project Integration

The complete project reference is available in the `project/` folder, showing how all concepts integrate into a cohesive application. Use this as a reference for understanding how individual exercises contribute to a larger system.

## Tips for Success

1. **Apply structured XML prompting** - Use the techniques from Lesson 3 throughout all exercises
2. **Maintain engineering ownership** - You make the decisions, AI provides assistance
3. **Focus on quality, not speed** - Good collaboration practices matter more than fast results
4. **Validate everything** - Test and review all AI-generated code
5. **Iterate systematically** - Use `<constraints>` and `<deliverables>` to refine outputs
6. **Document your process** - Note which prompt structures work best for different tasks

Remember: The goal isn't to see how well AI can code, but how well you can collaborate with AI to produce professional-grade software.
